# Миграция: Отслеживание выданных файлов по языку

## Описание проблемы
Клиенты могли выбрать один язык файла (русский или казахский), затем прокрутить вверх и нажать на другой язык, получив оба файла вместо одного.

## Решение
Добавлена система отслеживания выданных файлов:
1. В таблицу `order_items` добавлено поле `delivered_language` для отслеживания выданного языка
2. При выборе языка бот проверяет, не был ли уже выдан этот файл
3. После отправки файла кнопки выбора языка удаляются из сообщения

## Как применить изменения

### Шаг 1: Обновить базу данных в Supabase
Выполните SQL-миграцию в SQL Editor вашего проекта Supabase:

```sql
-- Добавить колонку для отслеживания выданного языка
ALTER TABLE public.order_items
ADD COLUMN IF NOT EXISTS delivered_language TEXT;

COMMENT ON COLUMN public.order_items.delivered_language IS 'Tracks which language variant was delivered: NULL (not delivered), ru, kz, or both';
```

### Шаг 2: Задеплоить код
После выполнения миграции задеплойте обновленный код:

```bash
# Если используете Vercel
vercel --prod

# Или через git push если настроен auto-deploy
git add .
git commit -m "Fix: Prevent duplicate file delivery for language selection"
git push origin main
```

### Шаг 3: Проверка
1. Создайте тестовый заказ с товаром, имеющим файлы на двух языках
2. Подтвердите заказ через админ-панель
3. В боте выберите один язык (например, русский)
4. Попробуйте нажать на кнопку казахского языка — должно появиться сообщение "⚠️ Этот файл уже был отправлен."
5. Кнопки должны исчезнуть после первого выбора

- `src/integrations-supabase/types.ts` - обновлены TypeScript типы
- `COMPLETE-SETUP.sql` - обновлена схема для новых установок
- `add-delivered-language-tracking.sql` - SQL-миграция для существующих баз

## Обратная совместимость
Для существующих заказов (где `delivered_language` будет NULL):
- Первый выбор языка сработает нормально и установит значение 'ru' или 'kz'
- Повторный выбор будет заблокирован
